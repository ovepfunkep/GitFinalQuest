# Финальный Git-квест «Калькулятор мечты»

Привет, я Иван. Добро пожаловать в маленький, но очень честный стартап, где ты — **стажёр**, а Git — твой главный инструмент выживания.

Задача этого квеста — не написать идеальный калькулятор (код у нас детсадовский), а показать, что ты умеешь:
- работать с ветками;
- смотреть историю;
- чинить свои же «улучшения» через revert/reset;
- не бояться merge‑конфликтов.

## Как стартовать

1. Склонируй этот репозиторий к себе локально.
2. Открой решение: `CalculatorApp/CalculatorApp.slnx` в Visual Studio.
3. Открой `CustomerTasks/README_TASKS.md` — там краткий обзор всех задач.
4. Начинай с `CustomerTasks/Task01/Chat.md` и двигайся по порядку до Task10.

## Общие правила

- **Одна задача — одна ветка.**  
  Для каждой задачи заводи свою ветку от актуального `main` (Игорь в чате подскажет названия).
- **Main не ломаем reset’ами.**  
  Reset используем только там, где Игорь прямо сказал, и только на своих локальных коммитах. Историю `main` не переписываем.
- **Подсказки — не туториал.**  
  В чатах Лера и Игорь говорят, **что** нужно сделать (создать ветку, сделать revert, посмотреть историю), но не расписывают пошагово, **куда кликать**. Конкретные названия кнопок смотри в `GitCheatsheet_VS.md`.
- **Git — главный герой.**  
  Код меняем только через готовые фрагменты из папки `Snippets/`, чтобы фокус оставался на Git‑операциях.

В конце квеста твой репозиторий будет оцениваться как контрольная: по структуре веток, истории коммитов, умению работать с историей и конфликтами. Поэтому читай чаты внимательно и не жди, что тебя будут вести за руку.
