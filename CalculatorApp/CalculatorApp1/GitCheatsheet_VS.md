# Git в Visual Studio — мини-шпаргалка

Эта страница — напоминалка, **где примерно искать** нужные действия в Visual Studio.  
Конкретные задачи и ветки смотри в чатах (`CustomerTasks/TaskNN/Chat.md`).

## Ветки

- **Создать новую ветку от текущей**
  - Открой панель Git (обычно снизу/справа).
  - Там есть список веток; рядом — кнопка создания новой ветки.
- **Переключиться на ветку**
  - В том же списке веток выбери нужную и активируй её (двойной клик / контекстное меню).
- **Переименовать ветку**
  - В списке веток по нужной ветке → контекстное меню → пункт переименования.
- **Удалить ветку**
  - Только после того, как она слита или больше не нужна: контекстное меню по ветке → удалить.

## Коммиты и история

- **Зафиксировать изменения (commit)**
  - Панель Git Changes: выбери файлы, напиши сообщение, нажми кнопку коммита.
- **Посмотреть историю ветки**
  - В панели Git или по проекту/решению через контекстное меню: пункт истории (History).
- **Сделать revert коммита**
  - В окне истории по нужному коммиту: контекстное меню → пункт revert.
- **Сделать reset ветки на более ранний коммит**
  - В окне истории по нужному коммиту: контекстное меню → пункт reset (используй только там, где задачи это разрешают).

## Несохранённые изменения

- **Отменить изменения в файле / всех файлах (discard/undo)**
  - В Git Changes по файлу (или по всему списку) есть действие для отката несохранённых правок.
  - Внимание: это удаляет незакоммиченные изменения без возможности вернуть.

## История конкретного метода (CodeLens)

- В C# файле над методом/классом Visual Studio показывает маленькую надпись:
  - кто менял, сколько раз, когда.
- Клик по этой надписи открывает список коммитов, которые трогали этот метод, и даёт перейти к diff.

Этих подсказок достаточно, чтобы ориентироваться по заданиям. Остальное — твоя внимательность и дружба с Git.
