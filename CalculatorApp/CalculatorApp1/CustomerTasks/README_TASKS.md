# Обзор задач заказчика

Ты — стажёр в стартапе, который пишет «Калькулятор мечты».  
Каждая задача — это маленький эпизод переписки Леры (заказчик) и Игоря (тимлид).

Задачи идут по порядку:

- **Task01** — первая версия калькулятора и первый нормальный коммит на `main`.
- **Task02** — базовые операции (+, -, *, /) в отдельной фича‑ветке и merge в `main`.
- **Task03** — смоделировать поломку деления и аккуратно отменить её через `revert`.
- **Task04** — экспериментальный шумный коммит и его откат через `reset` (только свой, локальный).
- **Task05** — найти нужную строчку в прошлом коммите и вернуть её руками.
- **Task06** — конфликт изменений в двух ветках и его решение в merge‑редакторе.
- **Task07** — переименовать неловкую ветку и убрать уже слитую.
- **Task08** — написать мусор в код и выбросить его, не попадая в историю (discard/undo changes).
- **Task09** — ветка с историей операций: несколько коммитов, частичный `revert`, merge в `main`.
- **Task10** — финальная уборка, красивый последний коммит и, при желании, тег версии.

Для каждой задачи:

- читай `CustomerTasks/TaskNN/Chat.md` — это основной сценарий;
- при ступоре можно заглянуть в `GitCheatsheet_VS.md` за напоминанием терминов;
- кодовые фрагменты (если есть) лежат рядом, в `CustomerTasks/TaskNN/`.

Двигайся последовательно: результат каждой задачи используется в следующей.
